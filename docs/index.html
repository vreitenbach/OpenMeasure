Why MeasFlow?

🚀

High Performance

Streaming-first architecture with zero-copy reads and memory-mapped I/O

🔓

Open & Free

MIT License with minimal dependencies - no vendor lock-in

🌐

Multi-Language

Native support for C#, Python, and C with consistent APIs

🚗

Automotive Ready

First-class support for CAN, CAN-FD, LIN, FlexRay, and Ethernet bus data

💾

Efficient Compression

Optional LZ4/Zstd compression for reduced storage

📊

Instant Statistics

Min/Max/Mean/StdDev computed during write, available instantly on read

Comparison with Existing Formats

Format Limitations
TDMS (NI) Proprietary, limited bus data support
HDF5 Complex C-based API, poor .NET interop
MDF4 (ASAM) Extremely complex spec, restricted tooling
MeasFlow ✅ Clean, open alternative with modern architecture

Simple API Example

// Write measurement data
using var writer = MeasFile.CreateWriter("data.meas");
var group = writer.AddGroup("Sensors");
var temp = group.AddChannel<float>("Temperature");

temp.Write(25.5f);
writer.Flush();

// Read measurement data
using var reader = MeasFile.OpenRead("data.meas");
var channel = reader["Sensors"]["Temperature"];
var stats = channel.Statistics;
Console.WriteLine($"Mean: {stats?.Mean}");

# Write measurement data
with meas.Writer("data.meas") as writer:
    group = writer.add_group("Sensors")
    temp = group.add_channel("Temperature", "float32")
    temp.write([25.5])
    writer.flush()

# Read measurement data
with meas.Reader("data.meas") as reader:
    channel = reader["Sensors"]["Temperature"]
    stats = channel.statistics
    print(f"Mean: {stats.mean}")

// Write measurement data
meas_writer_t* writer = meas_writer_create("data.meas");
meas_group_t* group = meas_writer_add_group(writer, "Sensors");
meas_channel_t* temp = meas_group_add_channel(group, "Temperature", MEAS_TYPE_FLOAT32);

float value = 25.5f;
meas_channel_write_float32(temp, &value, 1);
meas_writer_flush(writer);
meas_writer_destroy(writer);

// Read measurement data
meas_reader_t* reader = meas_reader_open("data.meas");
meas_channel_t* channel = meas_reader_get_channel(reader, "Sensors", "Temperature");
meas_statistics_t stats = meas_channel_get_statistics(channel);
printf("Mean: %f\n", stats.mean);
meas_reader_destroy(reader);

API Documentation

Comprehensive API reference for all three language bindings

Writer API

Method Description
MeasFile.CreateWriter(path) Create a new measurement file writer
writer.AddGroup(name) Add a new group to organize channels
group.AddChannel<T>(name) Add a typed channel (T = float, double, int, bool, etc.)
channel.Write(value) Write a single value to the channel
channel.Write(Span<T>) Write multiple values efficiently (zero-copy)
writer.Flush() Flush current data to a new segment on disk
writer.Dispose() Finalize and close the file (auto-called with using)

Reader API

Method Description
MeasFile.OpenRead(path) Open an existing measurement file for reading
reader[groupName] Access a group by name
group[channelName] Access a channel by name
channel.ReadAll<T>() Read all samples as ReadOnlySpan<T>
channel.ReadChunks<T>() Iterate through data chunks (streaming)
channel.Statistics Get pre-computed Min/Max/Mean/StdDev (instant, no I/O)
channel.Properties Access channel metadata (unit, description, etc.)

Bus Data API (CAN/LIN/FlexRay)

Method Description
writer.AddBusGroup(name, config) Add a bus channel (CAN, LIN, FlexRay, Ethernet)
busGroup.DefineCanFrame(name, id, length) Define a CAN frame with signals
frame.Signals.Add(...) Add signal definition (start bit, length, factor, etc.)
busGroup.WriteFrame(timestamp, id, data) Write raw frame bytes with timestamp
busGroup.DecodeSignal(signalName) Decode and extract signal values from frames

Example: Streaming Write

using var writer = MeasFile.CreateWriter("recording.meas");
var sensors = writer.AddGroup("Sensors");
var temp = sensors.AddChannel<float>("Temperature");

while (recording) {
    float[] batch = ReadFromSensor(batchSize: 10_000);
    temp.Write(batch.AsSpan());
    writer.Flush();  // Write segment, clear buffer
}

Writer API

Method Description
MeasWriter(path) Create a new measurement file writer (use with with statement)
writer.add_group(name) Add a new group to organize channels
group.add_channel(name, dtype) Add a channel with data type (e.g., MeasDataType.Float32)
channel.write(value) Write a single value or array/list to the channel
writer.flush() Flush current data to a new segment on disk
channel.properties[key] = value Set channel metadata

Reader API

Method Description
MeasReader(path) Open an existing measurement file for reading
reader[group_name] Access a group by name
group[channel_name] Access a channel by name
channel.read_all() Read all samples as NumPy array
channel.read_timestamps() Read timestamp channel as array of datetime objects
channel.statistics Get pre-computed statistics (count, min, max, mean, std_dev)
channel.properties[key] Access channel metadata
reader.groups Iterate over all groups in the file

Data Types

Type Description
MeasDataType.Int8 - Int64 Signed integers (8-64 bit)
MeasDataType.UInt8 - UInt64 Unsigned integers (8-64 bit)
MeasDataType.Float32, Float64 Floating point numbers
MeasDataType.Bool Boolean values
MeasDataType.Timestamp Nanosecond-precision timestamps
MeasDataType.Binary Variable-length binary frames

Example: Read and Analyze

with MeasReader("data.meas") as reader:
    motor = reader["Motor"]

    # Get instant statistics (no data read)
    stats = motor["RPM"].statistics
    print(f"RPM: {stats.min}-{stats.max}, avg={stats.mean}")

    # Read all data for analysis
    rpm_data = motor["RPM"].read_all()
    temp_data = motor["Temperature"].read_all()

    # Process with NumPy/Pandas
    import numpy as np
    correlation = np.corrcoef(rpm_data, temp_data)[0, 1]

Writer API

Function Description
meas_writer_open(path) Create a new measurement file writer
meas_writer_add_group(w, name) Add a new group, returns MeasGroupWriter*
meas_group_add_channel(g, name, type) Add a channel with data type (e.g., MEAS_FLOAT32)
meas_channel_write_f32(ch, data, count) Write float32 values (also _f64, _i32, etc.)
meas_channel_write_frame(ch, data, len) Write variable-length binary frame
meas_writer_flush(w) Flush current data to a new segment
meas_writer_close(w) Finalize and close the file

Reader API

Function Description
meas_reader_open(path) Open an existing measurement file
meas_reader_group_by_name(r, name) Get group by name, returns const MeasGroupData*
meas_group_channel_by_name(g, name) Get channel by name, returns const MeasChannelData*
meas_channel_read_f32(ch, buf, count) Read float32 values into buffer (also _f64, _i32, etc.)
meas_channel_next_frame(ch, state, data, len) Iterate through binary frames
ch->sample_count Total number of samples in channel
ch->has_stats, ch->stats Pre-computed statistics (min, max, mean, std_dev)
meas_reader_close(r) Close the reader and free resources

Example: Error Handling

MeasReader *r = meas_reader_open("data.meas");
if (!r) {
    fprintf(stderr, "Failed to open file\n");
    return 1;
}

const MeasGroupData *g = meas_reader_group_by_name(r, "Motor");
if (!g) {
    fprintf(stderr, "Group 'Motor' not found\n");
    meas_reader_close(r);
    return 1;
}

const MeasChannelData *rpm = meas_group_channel_by_name(g, "RPM");
if (rpm && rpm->has_stats) {
    printf("RPM range: %.1f - %.1f\n", rpm->stats.min, rpm->stats.max);
}

meas_reader_close(r);

Common Use Cases

📊 Streaming Data Acquisition

Record continuous sensor data with periodic flushes to disk, keeping memory usage constant regardless of recording duration.

🚗 Automotive Bus Recording

Capture CAN/LIN/FlexRay frames with frame definitions and signal decoding for automotive test automation.

📈 Post-Processing Analysis

Read measurements, access instant statistics, and process data with analysis tools (NumPy, Pandas, MATLAB).

🔄 Cross-Language Workflows

Write with C/C++ acquisition system, analyze with Python, visualize with C# tools—all using the same file format.

Streaming Architecture

MeasFlow is streaming-first — write and read data incrementally without holding entire files in memory

┌───────────────────┐
│  File Header 64B  │  Magic: MEAS\0, Version, GUID, Timestamp
├───────────────────┤
│  Metadata Segment │  Groups, channels, properties, bus definitions
│  → NextOffset ────┼──┐
├───────────────────┤  │
│  Data Segment #1  │◄─┘  First Flush() result
│  → NextOffset ────┼──┐
├───────────────────┤  │
│  Data Segment #2  │◄─┘  Second Flush() result
│  → NextOffset ────┼──┐
├───────────────────┤  │
│  Data Segment #N  │◄─┘  Final Dispose() data
└───────────────────┘

Bus Data Support (MDF4-compatible)

  • Bus types: CAN 2.0A/B, CAN-FD, LIN, FlexRay, Ethernet, MOST
  • Signal definitions: Start bit, bit length, Intel/Motorola byte order, factor/offset
  • PDU layer: AUTOSAR I-PDU with Container-PDU and Contained-PDU support
  • Multiplexing: MUX signals with nested multiplexing and range conditions
  • E2E Protection: AUTOSAR Profile 01-11, CRC/Counter positions
  • SecOC: Secure Onboard Communication with CMAC-AES/HMAC-SHA
